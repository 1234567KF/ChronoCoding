> 用于评测 AI 编程工具的 Token 效率

## 一、评判原则

### 1.1 核心原则

| 原则 | 说明 | 应用 |
|------|------|------|
| 控制变量 | 保持测试环境一致 | 仅改变被测工具 |
| 多维评估 | 从多个角度评估 | 效率+效果+质量 |
| 统计可靠 | 确保样本量足够 | 避免偶然性 |
| 可复现 | 测试可重复执行 | 标准化流程 |

### 1.2 避免偏差

```
常见偏差及避免:
├── 选择性偏差: 测试任务覆盖多种场景
├── 顺序效应: 随机化测试顺序
├── 环境差异: 使用Docker隔离环境
├── 主观评分: 使用客观指标为主
└── 数据泄露: 训练和测试数据分离
```

## 二、评测维度

### 2.1 一级维度

| 维度 | 权重 | 说明 |
|------|------|------|
| Token效率 | 35% | 压缩率和成本节省 |
| 任务完成 | 35% | 是否正确完成任务 |
| 稳定性 | 20% | 测试结果一致性 |
| 易用性 | 10% | 集成和配置复杂度 |

### 2.2 二级指标

```
Token效率:
├── 压缩率 = (原始 - 压缩) / 原始 × 100%
├── 节省Token数 = 原始 - 压缩
├── 成本节省 = 节省Token × 单价
└── 百万Token成本 ($/M)

任务完成:
├── 任务通过率 (Pass@1)
├── 完成时间 (Time)
├── 步数 (Steps)
└── 成功率 vs 基线

稳定性:
├── 方差 (Variance)
├── 变异系数 (CV)
└── 失败率

易用性:
├── 安装复杂度 (1-5)
├── 配置步骤数
└── 学习曲线
```

## 五、数据分析

### 5.1 基础统计

```python
def analyze_results(results):
    stats = {
        # 集中趋势
        "mean_savings": np.mean([r['savings_rate'] for r in results]),
        "median_savings": np.median([r['savings_rate'] for r in results]),
        
        # 离散程度
        "std_savings": np.std([r['savings_rate'] for r in results]),
        "min_savings": np.min([r['savings_rate'] for r in results]),
        "max_savings": np.max([r['savings_rate'] for r in results]),
        
        # 任务完成
        "pass_rate": sum(r['passed'] for r in results) / len(results),
    }
    return stats
```

### 5.2 对比分析

```python
def compare_tools(results_by_tool):
    comparison = {}
    for tool, results in results_by_tool.items():
        comparison[tool] = {
            "avg_savings": np.mean([r['savings_rate'] for r in results]),
            "pass_rate": sum(r['passed'] for r in results) / len(results),
            "avg_time": np.mean([r['time_ms'] for r in results]),
        }
    
    # 排名
    by_savings = sorted(comparison.keys(), 
                       key=lambda t: comparison[t]['avg_savings'], 
                       reverse=True)
    return comparison, by_savings
```

## 六、评分体系

### 6.1 综合评分公式

```python
def calculate_score(tool_results, baseline):
    # Token效率得分 (35%)
    token_score = tool_results['avg_savings'] * 35
    
    # 任务完成得分 (35%)
    # 计算相对于基线的完成率变化
    pass_diff = tool_results['pass_rate'] - baseline['pass_rate']
    task_score = (0.5 + pass_diff) * 35  # 假设完成率变化范围 [-0.5, 0.5]
    
    # 稳定性得分 (20%)
    # CV越低越好
    cv = tool_results['std_savings'] / tool_results['avg_savings']
    stability_score = max(0, (1 - cv)) * 20
    
    # 易用性得分 (10%)
    # 假设从外部配置获取
    usability_score = tool_results.get('usability', 5) * 2
    
    total = token_score + task_score + stability_score + usability_score
    return total
```

### 6.2 评分等级

| 等级 | 分数 | 说明 |
|------|------|------|
| A+ | 90-100 | 优秀，全面领先 |
| A | 80-89 | 很好，明显优势 |
| B+ | 70-79 | 良好，有优势 |
| B | 60-69 | 一般，持平 |
| C | 40-59 | 较差，有缺陷 |
| D | <40 | 不推荐 |

## 八、常见问题

### 8.1 如何处理异常值？

```python
# 方法1: 移除极端值
def remove_outliers(data, threshold=3):
    z_scores = np.abs(stats.zscore(data))
    return data[z_scores < threshold]

# 方法2: Winsorize (缩尾)
def winsorize(data, limits=(0.05, 0.05)):
    return stats.mstats.winsorize(data, limits=limits)
```

### 8.2 如何保证统计显著性？

from scipy import stats

def handle_failure(result):
    if result['error_type'] == 'timeout':
        return 'timeout'
    elif result['error_type'] == 'oom':
        return 'memory_error'
    elif result['error_type'] == 'api_error':
        return 'api_error'
    else:
        return 'unknown'
