## 三、分项评测结果

### 3.1 CLI 输出压缩（lean-ctx）⭐⭐⭐⭐⭐

| 命令类型 | 原始输出 | lean-ctx输出 | 压缩率 | 实测来源 |
|---------|---------|-------------|-------|---------|
| vitest run | 102,199 chars | 377 chars | **99.6%** | Terminal-Bench |
| npm test | 25,000 chars | 2,500 chars | **90%** | 社区反馈 |
| cargo test | 25,000 chars | 2,500 chars | **90%** | 社区反馈 |
| git status | 7,500 chars | 1,500 chars | **80%** | 社区反馈 |
| pytest | 20,000 chars | 5,000 chars | **75%** | 社区反馈 |

**结论**：lean-ctx 是本项目 token 节省效果最显著的单一工具，测试输出压缩率高达 90-99.6%

**行动项**：安装 lean-ctx（`lean-ctx init --agent claude`）以激活全部压缩能力

---

### 3.2 模型路由（kf-model-router）⭐⭐⭐⭐

| 场景 | Pro-only | Pro+Flash | 节省率 |
|------|---------|----------|-------|
| 计划阶段（5K in + 2K out） | $0.021 | $0.0036 | **83%** |
| 执行阶段（20K in + 8K out） | $0.084 | $0.0048 | **94%** |
| **综合** | $0.105 | $0.0084 | **92%** |

**质量验证原则**：
- 简单任务（搜索/格式化）→ Flash 直接执行
- 复杂任务（架构/评审）→ Pro 分析 → Flash 执行
- 关键任务（安全/审计）→ 保持 Pro

**结论**：kf-model-router 是模型级别的核心节省手段，自动化切换用户无感

---

### 3.3 智能调度（claude-code-pro）⭐⭐⭐⭐

| 场景 | 无CCP | 有CCP | 节省 |
|------|-------|-------|------|
| <3文件任务（跳过spawn） | 15K token | 0 | **跳过spawn，省15K** |
| ≥3文件任务（回调替代轮询） | 轮询~100次×500=50K | 回调0+确认1次 | **~97%** |

**判断逻辑**：
```javascript
shouldSkipSpawn = fileCount < 3 && !hasComplexDependencies
useCallback = agentSupportsCallback && estimatedPollCost > callbackOverhead
```

**结论**：ccp 在多 agent 场景下节省显著，回调替代轮询省 80-97%

---

### 3.4 通信压缩（lambda-lang）⭐⭐⭐

| 通信方式 | 单次token | 3次总token | 压缩率 |
|---------|----------|-----------|-------|
| 自然语言 | ~200 | ~600 | — |
| Lambda原子协议 | ~67 | ~200 | **66.5%** |

**结论**：lambda-lang 在多 agent 并发时自动生效，3x 压缩效果显著

---

### 3.5 代码审查（kf-code-review-graph）⭐⭐⭐

| 项目规模 | 全量扫描 | 依赖图谱 | 节省率 |
|---------|---------|---------|-------|
| 小（<10文件） | 10K token | 7K token | **30%** |
| 中（10-100文件） | 50K token | 20K token | **60%** |
| 大（100+文件） | 200K token | 60K token | **70%** |

**结论**：依赖图谱对中大型项目的节省效果更明显（60-70%）
