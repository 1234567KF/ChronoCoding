## 调用类型说明

| 调用类型 | 说明 | 例子 |
|---------|------|------|
| **自动调用** | 主技能执行时自动触发被调用技能，用户无感 | kf-spec → kf-model-router |
| **内部 spawn** | 技能执行时 spawn 子 Agent 并行工作 | /夯 → 红/蓝/绿队 Pipeline |
| **被动（被调用）** | 技能不自发执行，被其他技能通过 Hook 或 Agent 调用 | kf-web-search（被 /夯 调用） |
| **Hook 触发** | 技能执行到特定阶段时通过 Hook 触发另一个技能 | kf-prd-generator Hook → kf-alignment |
| **独立** | 不依赖也不调用其他技能（但可能被调用） | kf-go, kf-image-editor |

## 核心调用链（按入口）

### /夯（kf-multi-team-compete）完整调用链
```
用户: /夯 [任务]
  │
  ├─ kf-model-router → 裁判用 pro，各队用 flash
  │
  ├─ Pre-Stage（SDD Excel 时）
  │   └─ kf-prd-generator → PRD.md
  │       └─ kf-alignment（Hook）
  │
  ├─ Phase 1-4（红/蓝/绿三队并发）
  │   ├─ kf-alignment（对齐）
  │   ├─ kf-spec（Spec 生成）
  │   ├─ kf-ui-prototype-generator（UI 原型）
  │   ├─ kf-image-editor（配图）
  │   ├─ kf-web-search（查资料）
  │   ├─ kf-opencli（平台数据）
  │   ├─ kf-browser-ops（UI 测试）
  │   ├─ kf-code-review-graph（审查图谱）
  │   └─ gspowers Pipeline（阶段编排）
  │
  └─ Phase 4: 汇总融合 → 最终方案
```

### spec coding（kf-spec）完整调用链
```
用户: spec coding [需求]
  │
  ├─ Step 0: 技术选型
  │   └─ kf-model-router → pro
  │
  ├─ Step 1: 需求对齐
  │   └─ kf-alignment（自动调用）
  │
  ├─ Step 2-5: 生成 Spec 文档
  │   └─ 可选：kf-web-search（查资料）
  │
  ├─ Step 6: 分步编码
  │
  └─ 产出复盘
      └─ kf-alignment（自动调用）
```

### 逆向工程（kf-reverse-spec）完整调用链
```
用户: 逆向 [代码目录]
  │
  ├─ Stage 0: kf-alignment（对齐）
  ├─ Stage 1: 侦察（kf-web-search 查资料）
  ├─ Stage 2: 制图（kf-code-review-graph 依赖分析）
  ├─ Stage 3: 成文（Spec/Wiki）
  └─ Stage 4: 审查
```
