## 模板使用说明

### 1. settings.json.template（Claude Code 配置）

```powershell
# 复制到目标位置
Copy-Item templates\settings.json.template "$env:USERPROFILE\.claude\settings.json"

# 替换 Token 占位符
# 将 YOUR_TOKEN_HERE 替换为你的实际 Token
```

**必填字段**：
- `ANTHROPIC_AUTH_TOKEN`: 你的 API Token

**可选字段**：
- `model`: 模型名称（默认 MiniMax-M2.7）
- `theme`: 界面主题（light/dark）

### 2. config.yaml.template（ruflo 全局配置）

```powershell
# 创建目录
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude-flow"

# 复制配置
Copy-Item templates\config.yaml.template "$env:USERPROFILE\.claude-flow\config.yaml"
```

### 3. tdd-config.yaml.template（TDD 配置）

```powershell
# 创建目录
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude-flow"

# 复制配置
Copy-Item templates\tdd-config.yaml.template "$env:USERPROFILE\.claude-flow\tdd-config.yaml"
```

### 4. pre-commit.template（Git Hooks）

```bash
# 创建目录
mkdir -p ~/.claude-flow/hooks

# 复制并添加执行权限
cp templates/pre-commit.template ~/.claude-flow/hooks/pre-commit
chmod +x ~/.claude-flow/hooks/pre-commit
```

### 5. pipeline-example.md（Pipeline 配置示例）

此文件作为参考，复制到 `implementation-plan.md` 中的 `pipeline.modules` 部分即可使用。

### 6. wiki-template.md（Wiki 生成模板）

此文件作为参考，AI 在执行"生成 Wiki"时会参考此模板格式。

## 快速开始

### 在新环境执行

```
1. 将整个文件夹复制到新电脑
2. 打开终端，进入该文件夹
3. 让 AI 阅读 AI编程智驾手册.md
4. AI 自动完成所有安装和配置
5. 用户只需配置 Token（唯一需要手动介入的步骤）
```

### 完整执行流程

```
AI 读取 AI编程智驾手册.md
    ↓
AI 执行第八章 AI 执行清单（v1.3 完整版）
    ↓
├── 阶段一：环境准备（Git/Node.js）
├── 阶段二：核心安装（Claude Code/RTK/markitdown）
├── 阶段三：ruflo 安装（使用 templates/config.yaml.template）
├── 阶段四：gspowers 生态安装
├── 阶段五：技能扩充（用户说"允许 AI 自动扩充技能"后）
│   └── TDD 配置（使用 templates/tdd-config.yaml.template）
└── 阶段六：验证
    ↓
完成！用户可以开始使用
```

## 注意事项

1. **Token 配置**：唯一需要用户手动介入的步骤。使用 `templates/settings.json.template` 时记得替换 `YOUR_TOKEN_HERE`
2. **superpowers**：需要在 Claude Code 中执行 `/plugin install superpowers@claude-plugins-official`
3. **技能扩充**：用户需要说"允许 AI 自动扩充技能"才会安装 TDD/Wiki/Pipeline 扩展
4. **RTK**：需要 GitHub 网络访问，如遇问题请配置镜像源
5. **模板文件**：所有模板使用**相对路径**引用，确保在新环境中可直接使用
