## 技能分层

### Layer 0: 用户触发层
用户通过自然语言触发词调用技能，所有触发词在 `CLAUDE.md` 中注册。

| 触发词 | 技能 | 说明 |
|--------|------|------|
| `/go` | kf-go | 工作流导航 |
| `/夯 [任务]` | kf-multi-team-compete | 主入口，多团队竞争评审 |
| `/对齐` | kf-alignment | 对齐工作流 |
| `/prd-generator` | kf-prd-generator | PRD 文档生成 |
| `/web-search [问题]` | kf-web-search | 多引擎智能搜索 |
| `spec coding` | kf-spec | Spec 驱动开发 |
| `/review-graph` | kf-code-review-graph | 代码审查图谱 |
| `/browser-ops` | kf-browser-ops | 浏览器自动化测试 |
| `triple [任务]` | kf-triple-collaboration | 三方协作 |
| ... | ... | 完整列表见 AICoding.md |

### Layer 1: 模型路由层 (kf-model-router)
- **自动切换**：技能启动时通过 Hook 自动检查模型推荐
- **pro 场景**：架构设计、需求澄清、复杂 Bug、对齐工作流
- **flash 场景**：日常编码、代码审查、文档生成、执行阶段
- **用户无感**：所有切换自动完成

### Layer 2: 工作流编排层
| 技能 | 角色 | 特点 |
|------|------|------|
| kf-spec | Spec 驱动开发 | 6步 Pipeline：需求→设计→编码→测试 |
| kf-multi-team-compete | 多团队竞争评审 | 红/蓝/绿三队并发 + 裁判评分 + 汇总融合 |
| kf-reverse-spec | 逆向工程 | 存量代码→Spec/Wiki 文档 |
| kf-triple-collaboration | 三方协作 | 红队 vs 蓝队 vs 裁判 |
| kf-autoresearch | 自主 ML 实验 | 整夜自动改模型→训练→验证→循环 |
| kf-grant-research | 课题申报 | 查论文→分析→生成申报材料 |
| gspowers | SOP 导航 | /gspowers 查看进度和下一步 |

### Layer 4: gstack 产品流程框架
gstack 提供完整的产品开发流程框架，包含约 60 个子技能：
- **探索**：office-hours → design-consultation → plan-review 系列
- **构建**：design → design-html → browse（浏览器自动化）
- **审核**：codex → review → cso（安全审计）
- **发布**：ship → land-and-deploy → canary
- **维护**：retro → health → benchmark
