# 暗门注释规范 — 7 层结构化业务注释体系

> **目标**：PRD 与原型完全一致。人类可读（产品/开发/测试直接从注释核查业务逻辑），AI 可执行（仅凭 PRD+注释生成完整 Spec，无业务盲区）。

---

## 核心原则

1. **6 类注释全覆盖**：每个页面 MUST 包含业务描述、查询字段、列表字段、新增表单、修改表单、异常处理与边界值，按页面类型适配
2. **MECE 业务规则**：相互独立、完全穷尽，可直接转化为测试用例
3. **边界值显式标注**：每个输入字段 MUST 标注最大长度/数值范围等边界约束
4. **业务与技术分离**：注释只包含业务语义与数据约束，不混入技术栈选型
5. **易读无歧义**：表格格式统一，语义明确，无二义性
6. **按需可见**：注释角标仅在抽屉打开时显示，不影响页面原本视觉效果
7. **可调宽度**：抽屉宽度可通过拖拽左侧边缘调整（280px ~ 70vw）

---

## 6 类注释 → L0-L6 层映射

| 任务要求类型 | 映射层 | 说明 |
|-------------|--------|------|
| ① 业务描述 | **L0** 页面概述（增强版） | CRUD 矩阵 + 状态流转 + 数据权限 + 校验 + 关联 |
| ② 查询字段 | **L1-a** 查询字段字典 | 搜索区每个字段的独立规格 |
| ③ 列表字段 | **L1-b** 列表字段字典 | 表格列的独立规格 |
| ④ 新增表单字段 | **L1-c** 新增表单字段字典 | 创建表单每个字段的独立规格 |
| ⑤ 修改表单字段 | **L1-d** 修改表单字段字典 | 编辑表单每个字段 + 与新增的差异 |
| ⑥ 异常处理与边界值 | **L6** 测试场景（增强版） | 异常场景表 + 边界值定义表 |

## Layer 1 — 字段字典 (Field Dictionary) — 四段式

> L1 按字段使用场景拆分为 4 个独立子段：查询字段 / 列表字段 / 新增表单字段 / 修改表单字段。
> 每个子段独立标注，互不混淆。

---

### L1-a 查询字段字典

**每个查询字段** MUST 标注 4 属性：

| 属性 | 说明 | 必填 |
|------|------|------|
| 字段名 | 中文名称 + 英文 key | 是 |
| 字段类型 | string / number / date / daterange / enum / boolean | 是 |
| 前端表现形式 | 输入框 / 下拉框 / 日期选择器 / 日期范围 / 开关 | 是 |
| 数据校验规则 | 必填/格式/范围约束（如：最大长度 50，可模糊搜索） | 是 |

**格式模板：**
```
### 查询字段

| 字段名 | 字段类型 | 前端表现形式 | 数据校验规则 |
|--------|---------|-------------|-------------|
| {label} (`{key}`) | {type} | {widget} | {validation} |
```

**校验规则示例：**
| 字段 | 校验规则 |
|------|---------|
| 手机号 | 11 位数字，精确匹配 |
| 姓名 | 最大 20 字符，可模糊搜索 |
| 创建时间 | 日期范围，可选最近 7/30/90 天快捷选项 |
| 状态 | 枚举单选，无需校验 |

## Layer 2 — 业务规则 (Business Rules) — MECE

**定义规则：** 每条规则独立、互斥、组成完整集合，可直接映射到测试用例。

### 规则分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 校验规则 | 字段级和表单级校验 | "手机号必须为 11 位数字" |
| 计算规则 | 自动计算字段的公式 | "实付金额 = 订单金额 - 优惠金额 + 税费" |
| 联动规则 | 字段间的显示/值联动 | "选择'其他'时显示'其他原因'文本框" |
| 权限规则 | 基于角色的可见/可操作 | "仅管理员可执行退款操作" |
| 流程规则 | 状态流转的前置条件 | "已支付→已完成的必要条件是司机确认" |
| 幂等规则 | 重复操作的幂等性保证 | "同一订单 5 分钟内不可重复支付" |
| 超时规则 | 时间相关的自动行为 | "下单后 30 分钟未支付自动取消" |

### 格式模板（每条规则一个独立条目）

```
### BR-{序号} {规则名称}
| 属性 | 值 |
|------|-----|
| 类型 | 校验/计算/联动/权限/流程/幂等/超时 |
| 触发条件 | {condition} |
| 规则描述 | {description} |
| 违反时行为 | {violation_behavior} |
| 错误提示 | {error_message} |
| 测试用例 | TC-{序号}: {test_scenario} |
```

## 分页类型定制策略

不同页面类型的注释重点不同，避免信息过载：

### 列表页 (List Page)

| 层级 | 包含 | 权重 |
|------|------|------|
| L0 页面概述 | 完整（含 CRUD 矩阵 + 数据权限 + 关联） | 必选 |
| L1-a 查询字段 | 搜索区所有字段 | **核心** |
| L1-b 列表字段 | 表格列字段 | **核心** |
| L1-c 新增表单 | 弹出表单/抽屉表单字段 | **核心（若行内新增）** |
| L1-d 修改表单 | 编辑表单字段 + 与新增差异 | **核心（若有编辑功能）** |
| L2 业务规则 | 筛选逻辑 + 排序规则 + 分页规则 | **核心** |

### JS 切换逻辑

```javascript
var tabs = document.querySelectorAll('.anno-tab');
tabs.forEach(function(tab) {
  tab.addEventListener('click', function() {
    document.querySelectorAll('.anno-tab').forEach(function(t) { t.classList.remove('active'); });
    document.querySelectorAll('.anno-tab-content').forEach(function(c) { c.classList.remove('active'); });
    this.classList.add('active');
    document.getElementById(this.getAttribute('data-tab')).classList.add('active');
  });
});
```

### 分页类型 → Tab 层映射

| 页面类型 | L0 概述 | L1 字段 | L2 规则 | L3 状态 | L4 交互 | L5 设计 | L6 测试 |
|---------|---------|---------|---------|---------|---------|---------|---------|
| 列表页 | **必** | **核心**(a/b/c/d) | **核心** | 精简 | **核心** | 可选 | **必**(含异常+边界) |
| 表单页 | **必** | **核心**(c/d) | **核心** | — | **核心** | — | **必**(含异常+边界) |
| 详情页 | **必** | 必(b) | **核心** | **核心** | **核心** | — | **必**(含边界) |
| 仪表盘 | **必** | **核心**(b) | **核心** | — | **核心** | — | **必**(含边界) |

> **必** = 必须包含完整内容；**核心** = 重点信息密度最高；**精简** = 精简版内容；**可选** = 有则填，无则空；**—** = 页面不涉及，使用空占位文案
