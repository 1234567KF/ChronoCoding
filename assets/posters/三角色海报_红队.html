Role Breakdown
角色详解
协作者
Collaborator
特性亮点
  • Team Lead 统筹 — 先出计划、用户确认、再征召专家，避免盲目执行
  • 6 专家角色 — 前端/后端/QA/Code Review/调研/UX，各司其职
  • 三队并发竞争 — 红蓝绿队 Pipeline 六阶段，裁判择优融合
  • 用户介入三级 — 小改静默/中改通知/大改暂停，人机协同
  • 进化机制 — 个体经验积累 + 团队阵容优化，越用越强
  • Quality Signals — 标准化质量信号发射，Pipeline 各阶段自动检验
  • 条件重审 — P0/P1 缺陷密度检测，自动触发 reviewer 重审
技术升级路径
  • Qoder 专家团模式 → Team Lead + 多专家并行架构
  • gspowers SOP 导航 → Pipeline 六阶段流水线引擎
  • gstack 产品流程 → 从需求到交付的标准化框架
  • Lambda-lang A2A 协议 → Agent 间 3x 压缩通信
  • CCP 智能调度 → 回调替代轮询，省 80-97% 等待 Token
  • Gos 并发模型 (goroutine+channel) → 多 Agent 并发编排与通信
如何使用
  • 入口：/夯 [任务描述] 或 triple [任务]
  • 自动触发：被其他 kf- 技能按需 spawn（如 kf-reverse-spec 逆向工程）
  • 场景：新功能开发、复杂重构、代码评审、架构设计决策
  • 需主动触发：是 — 用户手动输入 /夯 启动
节流者
Saver / Throttler
特性亮点
  • KV Cache 优化 — DeepSeek 120x 差价杠杆，缓存命中 ¥0.025/M vs 未命中 ¥3/M
  • 共享前缀统一 — 所有 Agent prompt 前 200-500 token 逐字相同，最大化缓存复用
  • 模型智能路由 — 计划任务自动切 Pro，执行任务自动切 Flash，用户无感
  • 上下文压缩 — lean-ctx 90+ 压缩模式，CCP 智能跳过不必要的 Agent spawn
  • 成本预算控制 — 会话/任务级 Token 预算，超额自动降级
  • 节省追踪 (kf-saver) — 自动记录每次 API 调用的 token 节省数据，对比基线
  • 安全路由 — 密钥隔离 + 断路器 + 令牌桶限流 + 降级链，防连锁故障
技术升级路径
  • DeepSeek KV Cache (120x) → 共享前缀 + 预热策略 + TTL 管理，N agent 成本 ≈ 1+(N-1)×2%
  • lean-ctx (90+ 压缩模式) → CLI 输出省 80-99%，ctx_read 10 种模式，重读 ~13 tok
  • kf-model-router → Pro/Flash 分阶段路由，利用 3x 价差，成本降 ~92%
  • CCP/claude-code-pro → 跳过 spawn（省 15K tok/次）+ 回调替代轮询（省 80-97%）
  • Lambda-lang (340+ 原子) → Agent 间输出压缩 66.5%，A2A 通信 3x 压缩
  • claude-mem + context-mode → 跨会话记忆复用 + 会话连续性保缓存不中断
如何使用
  • 全自动运行：每次 API 调用自动压缩上下文、路由模型、优化缓存
  • 可选诊断：说 节省 / 缓存分析 / 成本优化 查看报告和建议
  • 场景：零感知后台优化，成本异常时自动告警
  • 需主动触发：否 — 所有核心节省机制全自动
监测者
Monitor
特性亮点
  • Token 全量追踪 — 每次 API 调用的 input/output/cache 全记录
  • 成本仪表盘 — 按模型/技能/Agent/会话维度实时呈现
  • 缓存命中监控 — /api/v1/cache 端点，命中率 < 30% 自动告警
  • 异常检测 — 缓存骤降 <20% · Token 暴涨 >200% · 模型比例偏离 >50%
  • Symphony 兼容 — REST API 标准接口，可接入外部监控体系
  • 重审触发检测 — review_reruns 表 + watcher 扫描 + API + 仪表盘，独立于 Agent 自报
  • Quality Signals — 标准化质量信号发射，自动接入监测者数据流
技术升级路径
  • Symphony 监控思想 → 统一 REST API 端点设计
  • Grafana 仪表盘模式 → Monitor Web 看板（3456 端口）
  • Graphify 依赖分析 → 技能调用链路追踪（skill_calls 表）
  • SessionStart/Stop Hook → 自动注入追踪，零侵入
如何使用
  • 入口：全自动（Hook 注入 SessionStart/Stop）
  • 手动触发：说 监测 / 面板 / 成本面板 / 缓存面板
  • 看板：浏览器访问 http://localhost:3456
  • 需主动触发：否 — 追踪全自动，看板/分析按需查看

→ claude-mem + context-mode
