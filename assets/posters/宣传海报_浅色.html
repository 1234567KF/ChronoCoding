· 常见问题 ·
QAI 编程智驾的 token 消耗会不会很贵？
A三层省 token 机制叠加：①模型路由（省原则）自动切换 pro/flash 模型匹配场景；②RTK（Rust Token Killer）拦截 CLI 命令输出，智能过滤压缩，终端操作综合节省约 80%；③context-mode会话压缩存活，压缩后自动恢复上下文，避免重复重建。三层叠加实际费用降到 1/5（省 5 倍的钱），长期 ROI 稳固。
✓ 成本可控
QAI 调研不准导致结果不准，怎么办？
A根因在于没有实时查询，查不到也查不全。我们采用多源验证机制（准原则）：web-search + scrapling 双引擎交叉确认，实时联网查证。从根源杜绝大模型"幻觉"，确保信息全面准确。
✓ 实时联网
Q单一 Agent 串行做事，不够专业且效率低？
A33 个 Agent 并发协作（夯原则），红蓝绿三队竞争评审，裁判择优融合。每个 Agent 专精特定领域（前端/后端/测试/安全/架构），并行推进需求、设计、编码、测试、审查全流程，博采众长、力大碾压。
✓ 多路并行
Q复杂任务很难跟 AI 交流清楚，怎么办？
A通过动前对齐（懂原则）先复述理解再执行，动后 Diff 确认偏差。Spec 驱动开发将模糊需求转化为结构化文档，智能拆解为可执行的子任务。全程可追溯、可确认，确保复杂任务零偏差传递。
