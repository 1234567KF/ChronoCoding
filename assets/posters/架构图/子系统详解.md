## 一、gstack 产品流程框架详解

gstack 是 Garry Tan 开发的产品流程框架，包含约 60 个子技能，覆盖从创意到部署运维的全流程。

### gstack 标准流程

```
创意 → 探索 → 规划 → 构建 → 审核 → 发布 → 运维
 │        │       │       │      │      │      │
 │    office-  plan-   design- review  ship   canary
 │    hours   ceo-    html            │      │
 │            review              land-  health
 │            │                   and-   │
 │            plan-               deploy retro
 │            eng-
 │            review
 │            │
 │            plan-
 │            design-
 │            review
 │            │
 │            plan-
 │            devex-
 │            review
```

### gstack 子技能分类

#### 探索与创意阶段
| 技能 | 说明 | 触发词 |
|------|------|--------|
| office-hours | YC 办公室时间——创业模式/创造者模式 | "brainstorm", "office hours" |
| design-consultation | 设计咨询——了解产品、调研竞品、提出完整设计系统 | "design system", "create DESIGN.md" |
| design-shotgun | 设计霰弹枪——生成多个设计变体、建立对比面板、收集反馈 | "explore designs", "show me options" |

#### 规划与评审阶段
| 技能 | 说明 | 触发词 |
|------|------|--------|
| plan-ceo-review | CEO 模式评审——重新思考问题、寻求 10x 产品、挑战前提 | "think bigger", "strategy review" |
| plan-eng-review | 工程经理模式评审——锁定执行方案、架构、数据流 | "engineering review", "lock in the plan" |
| plan-design-review | 设计师模式评审（计划阶段）——评分每项设计维度 | "review the design plan" |
| plan-devex-review | 开发者体验评审——竞品对标、设计魔法时刻 | "DX review", "devex review" |
| autoplan | 自动评审管线——依次运行所有评审技能，自动决策 | "auto review", "autoplan" |
| plan-tune | 计划调优 | — |

#### 构建与开发阶段
| 技能 | 说明 | 触发词 |

## 二、gspowers 技能详解

gspowers 是 AI 开发全流程编排器，作为 /夯 的 Pipeline 引擎被集成。

| 组件 | 说明 |
|------|------|
| **init** | 项目初始化流程 |
| **plan** | 计划制定和泳道分配 |
| **execute** | 任务执行编排 |
| **finish** | 完成和交付检查 |
| **sop** | 标准操作流程导航 |

gspowers 的技能文件位于 `skills/gspowers/`：
- `skills/gspowers/SKILL.md` — 核心技能定义
- `references/init.md` — 初始化参考
- `references/plan.md` — 计划参考
- `references/execute.md` — 执行参考
- `references/finish.md` — 完成参考
- `references/sop.md` — SOP 参考
