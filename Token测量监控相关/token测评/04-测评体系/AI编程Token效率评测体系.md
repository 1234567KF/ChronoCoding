> 版本：v1.0
> 生成时间：2026/05/05
> 对应专题任务步骤：步骤4

---

## 一、评测目标

评测 Token 节省效果，分维度量化，抛开价格波动，聚焦各环节节省率。

---

## 二、核心原则

1. **控制变量**：同一任务对比有/无某技能的结果
2. **多测求均值**：每次用例测 3 次，取平均值
3. **区分输入/输出**：输入和输出价格不同，必须分开记录
4. **区分命中缓存**：第1次、第2次、第3次结果可能因缓存而不同，需标注
5. **可复现**：测试用例需存档，结果可追溯

---

## 三、分项测试用例

### 用例1：kf-model-router（模型路由）

| 测试项 | 方法 |
|--------|------|
| **目标** | 验证 Pro 计划 + Flash 执行的质量等价性 |
| **方法** | 同一任务分别用 Pro-only vs Pro+Flash 路由，对比输出质量和 token 消耗 |
| **变量** | 模型选择（Pro vs Flash） |
| **常量** | 任务难度、输入内容、输出格式要求 |
| **指标** | ① 输出质量评分（1-10）② 输入 token ③ 输出 token ④ 总 token ⑤ 价格 |

**测试步骤**：
1. 准备3个标准任务（A/B/C 难度递增）
2. 用 Pro-only 执行，记录 token
3. 用 Pro+Flash（kf-model-router）执行，记录 token
4. 对比质量（人工或 LLM 评判）
5. 计算节省率 = (Pro-token - Pro+Flash-token) / Pro-token × 100%

---

### 用例2：claude-code-pro（智能调度 + 回调替代轮询）

| 测试项 | 方法 |
|--------|------|
| **目标** | 验证 <3 文件跳过 spawn 的准确性 + 回调替代轮询的节省 |
| **方法** | 对比有/无 CCP 的 agent 启动次数和轮询开销 |
| **变量** | 文件数量（1/2/3/5/10），有无 CCP |
| **常量** | 任务复杂度、Agent 类型 |
| **指标** | ① spawn 次数 ② 轮询次数 ③ 回调次数 ④ 总 token 消耗 |

**测试步骤**：
1. 准备不同文件数量的任务（1/2/3/5/10文件）
2. 无 CCP 执行，记录 spawn 和轮询 token
3. 有 CCP 执行，记录 spawn 和回调 token
4. 统计跳过次数（<3 文件应跳过 spawn）
5. 计算节省率

---

### 用例3：lambda-lang（3x 压缩）

| 测试项 | 方法 |
|--------|------|
| **目标** | 验证 Agent 间通信的压缩效果 |
| **方法** | 对比纯自然语言 vs Lambda 协议通信的 token 消耗 |
| **变量** | 通信方式（自然语言 vs Lambda 原子协议） |
| **常量** | 任务类型、通信次数 |
| **指标** | ① 单次通信 token ② 3次通信总 token ③ 压缩率 |

**测试步骤**：
1. 模拟 agent 间 3 次通信
2. 方式 A：纯自然语言（`请帮我执行 X`）
3. 方式 B：Lambda 原子协议（`!ta ct @task x`）
4. 对比 token 消耗，计算压缩率

---

### 用例4：kf-code-review-graph（依赖图谱）

| 测试项 | 方法 |
|--------|------|
| **目标** | 验证依赖图谱 vs 全量扫描的节省效果 |
| **方法** | 对比有/无图谱的代码审查 token 消耗 |
| **变量** | 项目规模（小/中/大）、有无图谱 |
| **常量** | 代码审查深度、审查范围 |
| **指标** | ① 输入 token ② 输出 token ③ 覆盖率 ④ 遗漏率 |

**测试步骤**：
1. 选择3个项目（小/中/大型各一）
2. 无图谱执行全量扫描，记录 token
3. 有图谱执行依赖树扫描，记录 token
4. 对比覆盖率（是否遗漏重要文件）
5. 计算节省率

---

### 用例5：RTK / lean-ctx（CLI 输出压缩）

## 六、综合评分体系

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| Token 节省率 | 40% | 各技能的综合节省率 |
| 质量等价性 | 30% | 节省后输出质量是否达标 |
| 稳定性 | 20% | 测试结果方差，稳定性高加分 |
| 易用性 | 10% | 用户无感知程度，自动化程度 |

### 综合节省率公式

```
综合节省率 = Σ(技能_i_节省率 × 技能_i_使用频率)
```
